**Overview**
- Geurts et al. (2009)-style entailment task: judge whether a Conclusion logically follows from a Premise.
- Purpose: test free-choice inferences under **want** (primary target) and (as a benchmark) under modals.
- Keys: B = FOLLOWS, N = DOES NOT FOLLOW.

**Design**
- Practice: 8 neutral logic trials with feedback (no modals/wants). These are hand-authored (and written out by `generate_stimuli.py`).
- Main: 96 trials (no feedback), with a midpoint break.
  - Critical: 48 trials (generated by `generate_stimuli.py`)
    - Modal: 24 (4 each) — allowed_and; allowed_single (counterbalanced A/B); must_and; must_single (counterbalanced A/B); all_and; all_single (counterbalanced A/B)
    - Want: 24 (4 each) — want_and; want_single (counterbalanced A/B); want_mind_and; want_mind_single (counterbalanced A/B); atleast_single_a; atleast_single_b
  - Controls/Fillers: 48 trials (hand-crafted in-code), 12 per category (conjunction, conditional, quantifier, presupposition), with each category containing 6 valid and 6 invalid items.

**Implementation**
- Code: `experiment/experiment.html` (single-screen presentation: Premise + Conclusion together; jsPsych + JATOS) and `generate_stimuli.py` (critical stimuli files for jsPsych + XLSX for review).
- Stimuli fields: `PREMISE`, `CONCLUSION`, `DOMAIN` (modal/want/control), `PAIR_TYPE`, `CONCLUSION_FORM`, `VALID` (Y/N/?), `ITEM`, `SCENARIO`, `TEMPLATE_ID`.
- Stimuli are written as JavaScript variable assignments and included via `<script src="...">`:
  - `var stimuli_experimental = [...]` in `experiment/stimuli_experimental.json` (critical + controls)
  - `var stimuli_practice = [...]` in `experiment/stimuli_practice.json` (hand-authored in-code)
- XLSX written at the root of this folder (`paper_analyses/Study1/`) (CSV fallback if no pandas).

**How to regenerate**
- Use any env with `pandas` + `openpyxl` (CSV fallback exists if pandas is missing).
- From this directory:
  - `python generate_stimuli.py` (full variety)
  - `python generate_stimuli.py --canonical-only` (first template per item type only)

**Notes**
- `generate_stimuli.py` builds the 48 critical trials and includes hand-crafted practice (8) and control/filler (48) items in-code.
- Action fillers are drawn from scenario bundles (e.g., cooking, photography, travel) to keep A/B internally coherent; all actions are unique across the 48 critical trials.
- Each item stores which template slot was used (`TEMPLATE_ID`); use `--canonical-only` to force template 1 for every type.
- Critical items currently mark validity as unknown (`VALID = ?`); adjust once definitive ground-truth labels are set.
- Items randomize names/objects and order. Within-participant, each premise appears with one conclusion only.
